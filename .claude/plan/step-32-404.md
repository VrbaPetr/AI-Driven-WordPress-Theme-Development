# Step 32 — Custom 404 Page

**Phase:** 7 — Forms & Site Utilities  
**Depends on:** Steps 01, 05, 06, 07  
**Required by:** nothing (end feature)

---

## Summary

Create a branded 404 error page with a clear headline, a helpful message, a search bar, and a link back to the home page. The design should feel intentional and on-brand, not like a generic server error.

---

## User Stories

- **As a visitor who lands on a broken link**, I want a helpful 404 page that tells me where I can go next so I don't immediately leave the site.
- **As a site owner**, I want the 404 page to match the theme's branding so the error experience feels professional rather than alarming.

---

## Business Value

A well-designed 404 page recovers a percentage of visitors who would otherwise bounce. Including search and key navigation links turns a dead end into a wayfinding opportunity.

---

## Acceptance Criteria

- [ ] `404.php` template created.
- [ ] Includes header and footer.
- [ ] Page header: large "404" number (decorative, `aria-hidden`), `<h1>` "Page Not Found" (or translated equivalent), one-sentence explanation.
- [ ] Search form for finding content.
- [ ] "Back to Home" button using the Button component.
- [ ] Optional: quick links to key sections (Services, Portfolio, Blog, Contact) — static links pointing to standard WP page paths or hardcoded common URLs.
- [ ] HTTP status code remains 404 (WordPress handles this automatically for `404.php`).
- [ ] All output escaped.
- [ ] `make check` passes.

---

| File | Change |
|---|---|

- [ ] Decorative "404" number is `aria-hidden="true"` — the `<h1>` "Page Not Found" is the meaningful heading
